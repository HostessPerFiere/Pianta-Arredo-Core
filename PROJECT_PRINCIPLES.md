# Project Principles

## Geometry First

Structured geometry is a primary source of truth.

## Domain Before UI

Interfaces consume the domain model. They do not define it.

## Specification Before Implementation

Stable concepts and public contracts should be defined before coding.

## Services Outside, Kernels Inside

Services expose public contracts. Kernels are replaceable implementations.

## Open Standards

Prefer documented and interoperable formats.

## Replaceable Components

Geometry, validation, knowledge, persistence and export components
must be replaceable through Ports and Adapters.

## No Vendor Lock-In

The Core must not depend on one cloud, database, CAD product or AI vendor.

## Event-Driven Where Useful

Events support plugins, observability and asynchronous workflows.

## Deterministic Core

Equal validated input should produce reproducible results.

## AI Is an Assistant

AI may suggest and interpret, but it must not silently replace
measurements, geometry, validation or professional review.

## Knowledge Is Data

Rules should be traceable and versioned instead of hidden in code.

## Testable by Design

Public Capabilities must define inputs, outputs and errors.

## Experimental Work Stays Outside the Core

Unproven ideas belong in RFCs, design documents or the playground.

## Professional Verification Remains Essential

The platform does not replace qualified architects, engineers,
surveyors or other authorised professionals.
